# StudyPilot Week 1 To Week 2 Comparison

This note compares the Week 1 and Week 2 StudyPilot milestones in instructional terms.

Comparison context:

* earlier version: `milestones/week01-basic-values`
* later version: `milestones/week02-decisions-and-loops`
* context: week-to-week milestone progression

## 1. Short Summary Of The Overall Difference

Week 1 is a first-program snapshot.

It shows one fixed study plan summary with assigned values and one simple subtraction expression.

Week 2 keeps StudyPilot small, but moves from one fixed summary to repeated processing.

It checks several study tasks, uses visible decision-making, and reports whether a weekly study goal has been met.

## 2. What Was Added

Week 2 adds these new ideas:

* a small fixed set of study tasks instead of one single task summary
* loop behavior so the program can process more than one task
* priority checks using comparisons
* completion checks using `if` and `else`
* running totals for planned and completed minutes
* a weekly-goal result at the end of the program

## 3. What Was Reorganized Or Improved

Week 2 improves the StudyPilot snapshot by making the logic more active.

Instead of only displaying assigned values, it now evaluates those values.

The program still keeps the logic direct and beginner-readable, but it now shows how repetition and decisions can make the output more useful.

This is a refinement of the Week 1 idea, not a different project.

## 4. What Is Still Intentionally Missing

Week 2 still keeps several later concepts out of scope:

* input prompts
* reusable functions
* larger state management patterns
* file saving and loading
* multi-file program structure
* APIs or framework concepts

Those features are still intentionally absent so the main lesson stays on decisions, comparisons, and loops.

## 5. Why This Version Is Appropriate For Its Place In The Sequence

Week 2 is appropriate because it adds exactly the kind of change the curriculum calls for:

* new concept added: conditionals and loop behavior
* refinement of prior concept: the fixed StudyPilot summary becomes repeated study-task processing
* concept still intentionally absent: broader application structure and persistence remain for later milestones

This keeps the milestone aligned with course readiness.

It gives students a visible reason for why Week 3 will need clearer organization through functions and more intentional collection use, without introducing those ideas too early.
