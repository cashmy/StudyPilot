# StudyPilot Week 7 To Week 8 Comparison

This note compares the Week 7 and Week 8 StudyPilot milestones in instructional terms.

Comparison context:

* earlier version: milestones/week07-rba-project-framing
* later version: milestones/week08-capstone-style-polish
* context: week-to-week milestone progression

## 1. Short Summary Of The Overall Difference

Week 7 focuses on project framing, constraints, structure choices, and accountable AI use.

Week 8 keeps that framing, but shifts toward polished presentation.

The instructional change is from explaining why the project was governed the way it was to presenting the full StudyPilot arc as a defensible, validated, and explainable reference set.

## 2. What Was Added

Week 8 adds these new ideas:

* a curated reference guide for the major StudyPilot versions
* a visible validation and revision note
* a presentation-oriented AI use and accountability note
* a more integrated view of the whole project rather than one framing layer

## 3. What Was Reorganized Or Improved

Week 8 improves the StudyPilot sequence by gathering design reasoning, validation evidence, and presentation framing into one polished inspection point.

Instead of focusing mainly on constraints and governance language, it packages the project so an instructor, reviewer, or student observer can understand what was built, how it was checked, and how it should be presented.

This is a refinement of the project explanation layer rather than a new implementation target.

## 4. What Is Still Intentionally Missing

Week 8 still keeps several things out of scope:

* unnecessary new features
* hidden framework expansion
* performative polish that reduces explainability
* a final surprise rebuild of the project

Those ideas are intentionally still missing so the main lesson stays on clarity, validation evidence, defensible design choices, and responsible AI-use explanation.

## 5. Why This Version Is Appropriate For Its Place In The Sequence

Week 8 is appropriate because it closes the milestone arc by emphasizing presentation quality without sacrificing explainability.

* new concept added: capstone-style integration, validation evidence, and presentation framing
* refinement of prior concept: Week 7 project framing is preserved, but now gathered into a more polished reference set
* concept still intentionally absent: unnecessary code growth and framework escalation remain out of scope

This fits the end of the sequence well.

After students and instructors have seen the project built, structured, and framed, Week 8 shows how the same work can be presented clearly and responsibly.
