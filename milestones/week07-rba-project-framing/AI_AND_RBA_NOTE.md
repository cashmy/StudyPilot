# AI And RBA Note

This note explains how StudyPilot connects to RBA-style project framing.

## Short RBA Connection

RBA means intent is framed first, AI works inside that frame, and outputs are checked against reality and revised when needed.

StudyPilot fits that pattern well.

## Where AI Can Help

AI can help with:

* drafting milestone code from a governed prompt
* generating comparison and explanation notes
* reorganizing code into clearer structures
* proposing documentation and teaching artifacts
* speeding up repetitive formatting or scaffolding work

## Where Human Judgment Must Govern

Human judgment must still decide:

* whether the milestone matches the intended course week
* whether the project is becoming too large or too advanced
* whether the artifact looks like a hidden student requirement
* whether the structure change is pedagogically useful
* whether a generated explanation is actually accurate

## Why Accountability Matters Here

Without human accountability, AI could push StudyPilot toward:

* premature complexity
* unnecessary features
* framework drift
* weak alignment with the course sequence

That would make the project less teachable, even if the code seemed more advanced.

## Practical Week 7 Lesson

The main Week 7 lesson is not "AI builds the project for us."

The lesson is that AI can accelerate drafting and revision, but the human still governs purpose, boundaries, and acceptance.

That is the connection between StudyPilot and Refraction-Based Architecture.
