# StudyPilot Milestone Index

This folder contains the weekly instructional StudyPilot snapshots plus one optional bridge milestone.

They are meant to be read in sequence so the project growth stays aligned with the course progression.

## Recommended Presentation Order

1. `week01-basic-values`
   Basic values, output, and simple expressions.
2. `week02-decisions-and-loops`
   Repeated processing, comparisons, and simple weekly-goal checks.
3. `week03-functions-and-collections`
   Functions, parameters, return values, and collection-based state.
4. `week04-debugging-testing-structure`
   Debugging process, expected-versus-actual checks, and pytest recognition.
5. `week05-files-and-persistence`
   JSON save/load behavior and simple error handling.
6. `week06-apis-and-app-architecture`
   Architecture recognition, separated responsibilities, and simulated JSON-style responses.
7. `week07-rba-project-framing`
   Project intent, constraints, structure choices, and accountable AI-use framing.
8. `week08-capstone-style-polish`
   Polished reference guidance, validation evidence, and presentation framing.

## Optional Bridge Before The Final Console App

* `bridge-final-integration`
   Code-bearing bridge milestone that closes most of the technical gap between Week 6 and the final console app while keeping the week numbering intact.

## How To Use This Folder

Use the milestone set when you want to show how StudyPilot grows one instructional layer at a time.

Use the optional bridge when you want an explicit technical integration step before moving from the milestones into the final plain-Python console reference in `full/01_console_app`.

Each folder includes:

* a README for the milestone boundary
* the milestone artifact set for that week
* comparison notes where useful to explain what changed from the previous week

## Boundary Reminder

These milestones are instructional snapshots.

They are not hidden student assignments, and they are not meant to replace the full reference versions in the `full` folder.
